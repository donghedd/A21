# 船舶设备故障维修知识图谱查询文档

## 1. 文档目的

本文档用于指导后续开发人员对接当前知识图谱数据库，统一查询方式、实体类型、关系类型和典型查询模板。

当前图谱基于 Neo4j，采用类型化节点和类型化关系，不再使用单一 `Entity/RELATED` 结构。

## 2. 图谱模型

### 2.1 节点类型

- `Book`
- `Device`
- `Fault`
- `Cause`
- `Symptom`
- `Action`
- `Parameter`

### 2.2 关系类型

- `COVERS`
- `HAS_FAULT`
- `CAUSED_BY`
- `HAS_SYMPTOM`
- `RESOLVED_BY`
- `TARGETS`
- `AFFECTS_PARAMETER`
- `SHOWS_AS`
- `HAS_COMPONENT`

### 2.3 典型知识链路

```text
Device -> HAS_FAULT -> Fault
Fault -> CAUSED_BY -> Cause
Fault -> HAS_SYMPTOM -> Symptom
Fault -> RESOLVED_BY -> Action
Fault -> AFFECTS_PARAMETER -> Parameter
Action -> TARGETS -> Device/Parameter
Parameter -> SHOWS_AS -> Symptom
Book -> COVERS -> 其他知识节点
Device -> HAS_COMPONENT -> Device
```

## 3. 节点与关系属性

### 3.1 节点属性

所有节点通常包含：

- `id`
- `name`
- `mentions`
- `source_books`

示例：

```cypher
MATCH (n:Fault)
RETURN n
LIMIT 5;
```

### 3.2 关系属性

所有关系通常包含：

- `weight`
- `source_book`
- `page`
- `evidence`

示例：

```cypher
MATCH ()-[r:CAUSED_BY]->()
RETURN r
LIMIT 5;
```

## 4. 基础查询

### 4.1 查询所有节点标签

```cypher
MATCH (n)
RETURN labels(n) AS labels, count(*) AS cnt
ORDER BY cnt DESC;
```

### 4.2 查询所有关系类型

```cypher
MATCH ()-[r]->()
RETURN type(r) AS relation_type, count(*) AS cnt
ORDER BY cnt DESC;
```

### 4.3 按名称精确查询节点

```cypher
MATCH (n {name: '跳闸'})
RETURN n, labels(n);
```

### 4.4 按标签和名称查询节点

```cypher
MATCH (n:Symptom {name: '跳闸'})
RETURN n;
```

### 4.5 模糊搜索节点

```cypher
MATCH (n)
WHERE n.name CONTAINS '跳闸'
RETURN n.name, labels(n), n.id
LIMIT 50;
```

### 4.6 多标签模糊搜索

```cypher
MATCH (n)
WHERE (n:Device OR n:Fault OR n:Cause OR n:Symptom OR n:Action OR n:Parameter)
  AND n.name CONTAINS '电压'
RETURN n.name, labels(n), n.id
LIMIT 100;
```

## 5. 邻居查询

### 5.1 查询某节点的一层邻居

```cypher
MATCH p = (n {name: '跳闸'})-[r]-(m)
RETURN p
LIMIT 200;
```

### 5.2 查询某节点的一层邻居明细

```cypher
MATCH (n {name: '跳闸'})-[r]-(m)
RETURN n.name AS source, type(r) AS relation, labels(m) AS target_labels, m.name AS target
ORDER BY relation, target
LIMIT 200;
```

### 5.3 查询两层邻居

```cypher
MATCH p = (n {name: '跳闸'})-[*1..2]-(m)
RETURN p
LIMIT 300;
```

### 5.4 查询指定关系类型的邻居

```cypher
MATCH p = (s:Symptom {name: '跳闸'})<-[r:HAS_SYMPTOM|SHOWS_AS]-(n)
RETURN p
LIMIT 200;
```

## 6. 设备相关查询

### 6.1 查询某设备的常见故障

```cypher
MATCH (d:Device {name: '发电机'})-[:HAS_FAULT]->(f:Fault)
RETURN d.name, f.name
ORDER BY f.name;
```

### 6.2 查询某设备的故障及其原因

```cypher
MATCH (d:Device {name: '发电机'})-[:HAS_FAULT]->(f:Fault)-[:CAUSED_BY]->(c:Cause)
RETURN d.name AS device, f.name AS fault, c.name AS cause
ORDER BY fault, cause;
```

### 6.3 查询某设备的故障、症状、处理动作

```cypher
MATCH (d:Device {name: '发电机'})-[:HAS_FAULT]->(f:Fault)
OPTIONAL MATCH (f)-[:HAS_SYMPTOM]->(s:Symptom)
OPTIONAL MATCH (f)-[:RESOLVED_BY]->(a:Action)
RETURN d.name AS device, f.name AS fault, collect(DISTINCT s.name) AS symptoms, collect(DISTINCT a.name) AS actions;
```

### 6.4 查询设备组成部件

```cypher
MATCH (d:Device {name: '液压系统'})-[:HAS_COMPONENT]->(c:Device)
RETURN d.name, c.name
ORDER BY c.name;
```

## 7. 故障相关查询

### 7.1 查询某故障的原因

```cypher
MATCH (f:Fault {name: '轴承损坏'})-[:CAUSED_BY]->(c:Cause)
RETURN f.name, c.name
ORDER BY c.name;
```

### 7.2 查询某故障的症状

```cypher
MATCH (f:Fault {name: '轴承损坏'})-[:HAS_SYMPTOM]->(s:Symptom)
RETURN f.name, s.name
ORDER BY s.name;
```

### 7.3 查询某故障的处理动作

```cypher
MATCH (f:Fault {name: '轴承损坏'})-[:RESOLVED_BY]->(a:Action)
RETURN f.name, a.name
ORDER BY a.name;
```

### 7.4 查询故障完整处理链

```cypher
MATCH (d:Device)-[:HAS_FAULT]->(f:Fault {name: '轴承损坏'})
OPTIONAL MATCH (f)-[:CAUSED_BY]->(c:Cause)
OPTIONAL MATCH (f)-[:HAS_SYMPTOM]->(s:Symptom)
OPTIONAL MATCH (f)-[:RESOLVED_BY]->(a:Action)
OPTIONAL MATCH (f)-[:AFFECTS_PARAMETER]->(p:Parameter)
RETURN d.name AS device,
       f.name AS fault,
       collect(DISTINCT c.name) AS causes,
       collect(DISTINCT s.name) AS symptoms,
       collect(DISTINCT p.name) AS parameters,
       collect(DISTINCT a.name) AS actions;
```

## 8. 症状相关查询

### 8.1 查询某症状对应的故障

```cypher
MATCH (f:Fault)-[:HAS_SYMPTOM]->(s:Symptom {name: '跳闸'})
RETURN f.name
ORDER BY f.name;
```

### 8.2 查询某症状相关的参数异常

```cypher
MATCH (p:Parameter)-[:SHOWS_AS]->(s:Symptom {name: '跳闸'})
RETURN p.name
ORDER BY p.name;
```

### 8.3 查询症状到故障到原因链路

```cypher
MATCH (f:Fault)-[:HAS_SYMPTOM]->(s:Symptom {name: '跳闸'})
OPTIONAL MATCH (f)-[:CAUSED_BY]->(c:Cause)
RETURN s.name AS symptom, f.name AS fault, collect(DISTINCT c.name) AS causes
ORDER BY fault;
```

### 8.4 查询症状相关图谱子图

```cypher
MATCH p = (s:Symptom {name: '跳闸'})
-[:HAS_SYMPTOM|SHOWS_AS|AFFECTS_PARAMETER|CAUSED_BY|RESOLVED_BY|HAS_FAULT|TARGETS*1..3]-
(n)
RETURN p
LIMIT 300;
```

## 9. 维修动作相关查询

### 9.1 查询某动作处理哪些故障

```cypher
MATCH (f:Fault)-[:RESOLVED_BY]->(a:Action {name: '调整电源电压'})
RETURN f.name
ORDER BY f.name;
```

### 9.2 查询某动作作用于哪些对象

```cypher
MATCH (a:Action {name: '调整电源电压'})-[:TARGETS]->(n)
RETURN a.name, labels(n), n.name
ORDER BY n.name;
```

### 9.3 查询动作相关的 2 层图谱

```cypher
MATCH p = (a:Action {name: '调整电源电压'})-[*1..2]-(n)
RETURN p
LIMIT 300;
```

### 9.4 查询动作相关的 7 层图谱

```cypher
MATCH p = (a:Action {name: '调整电源电压'})
-[:TARGETS|RESOLVED_BY|HAS_FAULT|CAUSED_BY|HAS_SYMPTOM|AFFECTS_PARAMETER|SHOWS_AS|HAS_COMPONENT|COVERS*1..7]-
(n)
RETURN p
LIMIT 300;
```

## 10. 参数相关查询

### 10.1 查询某参数对应哪些症状

```cypher
MATCH (p:Parameter {name: '电压过低'})-[:SHOWS_AS]->(s:Symptom)
RETURN p.name, s.name;
```

### 10.2 查询哪些故障影响某参数

```cypher
MATCH (f:Fault)-[:AFFECTS_PARAMETER]->(p:Parameter {name: '电压过低'})
RETURN f.name
ORDER BY f.name;
```

### 10.3 查询参数异常相关处理链

```cypher
MATCH (f:Fault)-[:AFFECTS_PARAMETER]->(p:Parameter {name: '电压过低'})
OPTIONAL MATCH (f)-[:CAUSED_BY]->(c:Cause)
OPTIONAL MATCH (f)-[:RESOLVED_BY]->(a:Action)
RETURN p.name AS parameter, f.name AS fault, collect(DISTINCT c.name) AS causes, collect(DISTINCT a.name) AS actions;
```

## 11. 书籍与证据追溯查询

### 11.1 查询某书覆盖的知识节点

```cypher
MATCH (b:Book {name: '船舶电气设备维修指南'})-[:COVERS]->(n)
RETURN labels(n), n.name
LIMIT 200;
```

### 11.2 查询某关系的证据

```cypher
MATCH (f:Fault {name: '轴承损坏'})-[r:CAUSED_BY]->(c:Cause)
RETURN f.name, c.name, r.source_book, r.page, r.evidence;
```

### 11.3 查询某节点全部相关证据

```cypher
MATCH (n {name: '跳闸'})-[r]-(m)
RETURN n.name, type(r) AS relation, m.name, r.source_book, r.page, r.evidence
LIMIT 200;
```

## 12. 路径查询

### 12.1 查询两个节点之间的最短路径

```cypher
MATCH (a {name: '发电机'}), (b {name: '跳闸'})
MATCH p = shortestPath((a)-[*..6]-(b))
RETURN p;
```

### 12.2 查询设备到处理动作的路径

```cypher
MATCH (d:Device {name: '发电机'}), (a:Action {name: '调整电源电压'})
MATCH p = shortestPath((d)-[:HAS_FAULT|CAUSED_BY|HAS_SYMPTOM|RESOLVED_BY|TARGETS|AFFECTS_PARAMETER|SHOWS_AS*..6]-(a))
RETURN p;
```

### 12.3 查询症状到设备的路径

```cypher
MATCH (s:Symptom {name: '跳闸'}), (d:Device {name: '发电机'})
MATCH p = shortestPath((s)-[*..6]-(d))
RETURN p;
```

## 13. 前端推荐查询

### 13.1 搜索接口推荐查询

```cypher
MATCH (n)
WHERE n.name CONTAINS $keyword
RETURN n.id AS id, n.name AS name, labels(n)[0] AS label
LIMIT 20;
```

### 13.2 节点详情页推荐查询

```cypher
MATCH (n {id: $id})
OPTIONAL MATCH (n)-[r]-(m)
RETURN n,
       collect(DISTINCT {
         relation: type(r),
         target_id: m.id,
         target_name: m.name,
         target_label: labels(m)[0],
         source_book: r.source_book,
         page: r.page,
         evidence: r.evidence
       }) AS neighbors;
```

### 13.3 图谱浏览页推荐查询

```cypher
MATCH p = (n {id: $id})-[*1..2]-(m)
RETURN p
LIMIT 200;
```

### 13.4 故障专题页推荐查询

```cypher
MATCH (f:Fault {id: $id})
OPTIONAL MATCH (d:Device)-[:HAS_FAULT]->(f)
OPTIONAL MATCH (f)-[:CAUSED_BY]->(c:Cause)
OPTIONAL MATCH (f)-[:HAS_SYMPTOM]->(s:Symptom)
OPTIONAL MATCH (f)-[:RESOLVED_BY]->(a:Action)
OPTIONAL MATCH (f)-[:AFFECTS_PARAMETER]->(p:Parameter)
RETURN f,
       collect(DISTINCT d.name) AS devices,
       collect(DISTINCT c.name) AS causes,
       collect(DISTINCT s.name) AS symptoms,
       collect(DISTINCT a.name) AS actions,
       collect(DISTINCT p.name) AS parameters;
```

## 14. 管理与校验查询

### 14.1 统计各标签节点数

```cypher
MATCH (n)
RETURN labels(n)[0] AS label, count(*) AS cnt
ORDER BY cnt DESC;
```

### 14.2 统计各关系类型数量

```cypher
MATCH ()-[r]->()
RETURN type(r) AS type, count(*) AS cnt
ORDER BY cnt DESC;
```

### 14.3 检查孤立节点

```cypher
MATCH (n)
WHERE NOT (n)--()
RETURN labels(n), n.name, n.id
LIMIT 100;
```

### 14.4 清空当前数据库

```cypher
MATCH (n)
DETACH DELETE n;
```

## 15. 查询注意事项

### 15.1 命名注意

- 节点查询优先使用 `id`
- 按 `name` 查询时注意同名风险
- 前端不建议直接拼接 Cypher，建议参数化查询

### 15.2 性能注意

- 图谱页面默认只查 1~2 层
- 最短路径建议限制最大跳数
- 图可视化查询要加 `LIMIT`
- 搜索和图扩展应分开接口

### 15.3 展示注意

- `COVERS` 适合做证据追溯，不建议默认在主图中全部展开
- 故障诊断页面优先展示 `HAS_FAULT / CAUSED_BY / HAS_SYMPTOM / RESOLVED_BY`
- 参数异常相关展示优先用 `AFFECTS_PARAMETER / SHOWS_AS`

## 16. 推荐开发规范

### 16.1 API 参数化

推荐后端统一采用参数化查询，例如：

```cypher
MATCH (n {name: $name})
RETURN n;
```

### 16.2 图查询层数控制

建议后端对扩展层数做白名单限制：

- 默认 `1`
- 最大 `3`
- 特殊分析场景最大 `7`

### 16.3 返回结构建议

前端图谱接口建议统一返回：

- `nodes`
- `edges`
- `meta`

其中：

- `nodes` 包含 `id/name/label`
- `edges` 包含 `source/target/type/evidence`
- `meta` 包含统计信息

## 17. 当前推荐优先使用的查询模板

如果后续开发时间紧，优先接这几类：

1. 搜索节点
2. 节点详情
3. 一层/两层邻居
4. 设备 -> 故障 -> 原因 -> 症状 -> 动作
5. 症状反查故障
6. 证据追溯

这几类基本可以支撑一个可用的网页系统。
